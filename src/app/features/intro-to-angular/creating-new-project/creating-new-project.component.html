Starting your first project

When you complete the below steps check out the CLI documentation. Start with the documentation for generate. Even if you are lost the most important thing is to begin the process of familiarizing yourself with it. So if you do have a problem or forget the syntax you can go to the source. A lesser known page are the Stories located within the angular-cli github repo. Take a look at the titles and make a mental note of what is located here. Most likely you will find one or more of these useful in the future.

First things first. Let's generate our project! For your reference - ng new documentation

  • Open VSCode
  • Open terminal ()

Let's break it down.

  • cd c:\git
    • cd = change directory. This is a windows command. If you are on a Mac or Linux you can google "How to change directory from command line on Mac".
  • ng new MyAwesomeProject --routing=true --style=scss
    • ng new MyAwesomeProject
      • Creates a new project called MyAwesomeProject.
    • --routing=true
      • Tells the Angular CLI to generate the appropriate files for routing/navigation. We will review routing in more detail later.
    • --style=scss
      • Tells the Angular CLI to use Sass instead of the default css. Don't worry if you don't know what Sass is, you can type css directly into the .scss files. The reason I like to use Sass instead of sticking with the default is because adding scss later is somewhat of a pain and even if you don't use any Sass features it doesn't hurt to have it.

Setup Core and Shared Modules

In this section we are going to generate two important modules. I'll give a brief description here but both modules deserve their own page with definitions and examples. When I first started learning Angular the purpose and use of these modules eluded me for some time, so I hope to bring some clarity early on to you. However if you leave still scratching your head then don't worry, this is normal. In time they will make more and more sense.

At this point we haven't even defined what a Module is. So don't worry about absorbing these definitions.

  • Core Module
    • The place we put everything we normally would put in the App Module.
    • Wait? Are you serious? Yes, seriously the Core module keeps the App module uncluttered. Clutter = bad.
  • Shared Module
    • The place we put generic stuff used accross our entire application.
Using the shorthand notation for commands is much more efficient.
  • For example
    • ng g m shared  ==  ng generate module shared
