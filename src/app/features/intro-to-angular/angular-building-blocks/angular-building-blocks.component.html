You put these blocks together to build a Single Page Application . If you are coming from an AngularJS background you will be familiar with most of these terms. Most Common

An (Angular) Module, otherwise known as NgModule, is the basic organization block of angular. NgModules glue your app together and give your application a virtual structure. Another useful way of thinking about NgModules is a container. For example, if you create a component (or whatever) you will access that component through the NgModule you will not directly reference the component. More on this later. Also be sure to not confuse NgModules with Typescript Modules or Javascript Modules. That isn't say Angular doesn't use the Module system, it absolutely does! Take a look below at the CoreModule for Omg, It's Angular! application.

Don't get caught up in all the details of an NgModule. We will cover the inner workings of an NgModule later. The key takeaway is NgModules give us a way to organize our application.
